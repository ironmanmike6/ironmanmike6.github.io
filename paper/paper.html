Amazon's Tech Stack

Server Scale

Amazon's server stack is vast and complex. It is not an easy task to simply explain what they use for particular tasks, because they simple answer is, they use a multitude of different types of servers for each task. Luckily, the servers that Amazon uses to host their website are the very same ones that they make available to other companies such as Netflix and Autodesk on their Amazon Web Services program.

Amazon Web Services (AWS) is a massive assortment of compute, storage, database, analytics, application and deployment services. Essentially, anything you might need a server or servers for, Amazon is able to provide. These services also have to ability to scale massively with the help of features such as Elastic Load Balancing and Elastic Cloud Computing (EC2). These features combined allow anyone, including Amazon themselves to scale up or down the number of servers dedicated to any one task. To show just how ridiculous these servers can scale, Netflix uses 35% of the bandwidth in North America during peak hours. Netflix is hosted entirely on Amazon's servers using Amazon Web Services.

The Actual Tech

OS Choices

Amazon runs virtually everything on Linux machines. Primarily they run either RHEL5 or a specific Amazon Linux distribution.

Platforms

Amazon uses a variety of languages, but primarily uses C++, Perl, Mason, Java, Jboss and Servlets. The website makes hundreds of calls to various services on the backend to display a single web page. Most third party technologies don't scale to the massive size that Amazon expects and thus many features are created, a lot of which are available to the public through AWS.

Computational Servers

Amazon has a series of computationally focused servers which has the c4.large which as 2 virtual CPUs, and 3.75 GiB of memory. There a series of larger and larger computational servers all the way up to the C4.8xlarge which has 36 virtual CPUs and 60 GiB of memory. The storage on these servers is done through Elastic Block Storage which stores everything on a remote server and keeps data safe. These servers feature a high frequency Intel Xeon E5-2666 v3 (Haswell) processor.

Development Servers

The smallest server Amazon uses is the t2.micro which has only a single CPU, and 1 GiB of memory. They also offer t2.small and t2.medium which have 1 CPU with 2 GiB of memory and 2 CPUs with 4 GiB of memory respectively. These run on Intel Xeon Processors from 2.5GHz to 3.3GHz with Turbo.

Database Servers

Amazon has many high storage servers such as the i2.xlarge which features 4 virtual CPUs, 30.5 GiB of memory and 1x800 GB SSD. The largest storage optimized server is the i2.8xlarge which as 32 virtual CPUs, 244 GiB of memory and 8x800 GB SDD. It runs on an Intel Xeon E5-2670 v2 (Ivy Bridge) processor and has high random I/O performance.

Data Warehouse

Amazon has one server specifically optimized for high reads and writes performance with very high storage. It is the hs1.8xlarge and features 16 virtual CPUs, 117 GiB of memory and 24x2000 GB of storage. It is capable of 2.6 GB per second read and write performance with 2 MiB block size.

Security

Amazon employs a variety of methods of security, with the most important one being on how they handle users personal information such as credit cards and addresses. Amazon uses the Netscape Secure Commerce Server using the SSL protocol. Credit cards are stored in a database not accessible from the internet. Amazon allows users to encrypt every piece of information that they enter about themselves.
